# E220 Configurator - Design Context

## Product Purpose
E220 LoRa Module Configurator is a professional hardware configuration tool for embedded systems engineers and IoT developers. It enables configuration of EBYTE E220-900T30S LoRa modules via binary register protocol.

## Target Audience
- **Primary**: Embedded systems engineers, IoT developers, LoRa enthusiasts
- **Context**: Linux/Raspberry Pi environments, often used in production/field settings
- **Expertise Level**: Technical, hardware-comfortable
- **Settings**: Both desktop (workbench) and headless (SSH over network) usage

## Use Cases
1. **Initial Module Setup** - Configure frequency, baud rate, power, addressing for new modules
2. **Parameter Tuning** - Adjust air rate, transmission power, WOR settings for specific applications
3. **Debugging** - Read current parameters, verify register values, troubleshoot connectivity
4. **Batch Configuration** - Load/save configurations as JSON files for fleet management
5. **Real-time Testing** - Send/receive test data to verify wireless links

## Brand Personality / Tone
**Aesthetic Direction**: Industrial/Utilitarian with Technical Refinement

The interface should feel:
- **Purposeful & Direct**: No fluff, every element has function
- **Professional**: Trust-inspiring, suited for production environments
- **Technical but Accessible**: Complex parameters explained clearly, not dumbed down
- **Data-Driven**: Values, frequencies, register states shown clearly
- **Dark-Friendly**: Works well in dim workbench environments
- **Responsive to State**: Visual feedback for connection status, write success, errors

## Key Differentiators
What makes this memorable:
1. **Clear Hierarchy**: Advanced parameters grouped/hidden by default, revealed through progressive disclosure
2. **Live Frequency Display**: Show calculated frequency in real-time as user adjusts channel
3. **Register Visualization**: Show actual 8-byte register values being sent/written
4. **Connection State**: Prominent, always-visible connection indicator with detailed status messages
5. **Intentional Color**: Use LoRa frequency bands (900 MHz region) inspired palette (warm amber/gold for 900MHz band)
6. **Technical Typography**: Monospace for register values, clean sans-serif for labels

## Design Constraints
- **Framework**: Python tkinter (desktop application, not web)
- **Environment**: Linux/Raspberry Pi (may be accessed remotely over SSH)
- **Performance**: Lightweight, no heavy animations (embedded systems context)
- **Accessibility**: High contrast, keyboard-navigable, clear focus states
- **Offline**: Works without internet, pure serial communication

## Visual Direction Summary
- **Palette**: Warm amber/gold accents on dark background (900 MHz inspired), with teal secondary (frequency complementary)
- **Typography**: IBM Plex Mono for registers/values, Inter/Roboto for UI labels (both available via tkinter)
- **Layout**: Left sidebar navigation, main content area with tabs, right info panel for real-time status
- **Emphasis**: Connection status always visible, frequency/channel live display, register byte visualization
- **Spacing**: Generous whitespace around functional areas, tight grouping of related parameters
- **Motion**: Minimal (tkinter constraints), but smooth state transitions and error feedback

## Design Principles
1. **Single Source of Truth**: Show register values, not interpreted abstractions (users can verify hardware)
2. **Progressive Disclosure**: Basic mode by default, click to expand advanced options
3. **Real-Time Feedback**: Frequency, calculated values update as user types
4. **Error Prevention**: Invalid ranges highlighted, CLI examples provided for every parameter
5. **Power User Support**: Keyboard shortcuts, command-line alternative, batch operations via JSON

## Success Metrics
- User can configure module parameters in < 2 minutes
- Register values are auditable (user can verify what was written)
- No need to consult external documentation for parameter ranges
- Works equally well on 24" desktop monitor and 15" laptop screen
- Professional appearance (not generic tkinter, not "AI slop")
