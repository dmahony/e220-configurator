INTERACTION PATTERNS
====================

Connection Flow:
  1. User selects serial port from dropdown
  2. Clicks amber "Connect" button
  3. Button changes to "Disconnect" (red)
  4. Status indicator turns green
  5. Other tabs become enabled
  6. User can now read/write settings

Configuration Flow:
  1. User adjusts parameters (sliders, dropdowns, checkboxes)
  2. Values update in real-time (e.g., frequency display)
  3. Right panel shows current values
  4. Click "Save Config" to write to hardware
  5. Feedback: "Parameters written successfully"
  6. Register display shows actual hardware state

Debugging Flow:
  1. User clicks "Monitor" tab
  2. Register values display as hex dump
  3. Click "Refresh Registers" to update
  4. Can verify what was written to hardware
  5. Each byte corresponds to specific settings

DESIGN PHILOSOPHY
=================

This interface embodies Impeccable's design principles:

1. BOLD AESTHETIC DIRECTION
   - Committed to industrial/utilitarian look
   - Not generic or templated
   - Intentional design choices throughout

2. NO "AI SLOP"
   - Avoids overused fonts (no Inter, Arial, defaults)
   - No glassmorphism or blur effects
   - No cards nested in cards
   - No gradient text or bounce animations
   - No sparklines as decoration

3. CONTEXT-AWARE DESIGN
   - Dark theme for workbench/LED environments
   - Warm amber inspired by 900 MHz frequency band
   - Teal complement for frequency/technical indicators
   - Industrial aesthetic for embedded engineers

4. REAL-TIME FEEDBACK
   - Frequency updates as user adjusts channel
   - Connection status always visible
   - Register values available for verification
   - Quick info panel shows current state

5. PROGRESSIVE DISCLOSURE
   - Basic settings visible by default
   - Advanced options in separate tab
   - Quick actions immediately accessible
   - Detailed debugging in monitor tab

6. TECHNICAL PRECISION
   - Monospace font for register values
   - Hex display for hardware verification
   - Parameter ranges shown in hints
   - Clear units and measurement indicators

ANTI-PATTERNS AVOIDED
====================

✗ Overused fonts       → Custom typography hierarchy
✗ Gray on colored BG   → Text always on dark background
✗ Cards in cards       → Flat, section-based layout
✗ Glassmorphism        → Clean, flat design
✗ Bounce animations    → Smooth, instant transitions
✗ Gradient text        → Simple, readable text
✗ Sparklines           → Only meaningful data
✗ Modals               → Inline dialogs and forms
✗ Identical card grids → Varied layouts and groupings
✗ Pure black/white     → Tinted colors (#0f0f0f, #f5f5f5)
